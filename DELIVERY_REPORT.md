## 📊 项目概览

### 核心功能
一个基于 AI 的漫画生成工具，用户只需输入一句话创意，系统将自动完成：
1. 剧本创作（AI 编剧）
2. 角色设计（角色设定图）
3. 分镜绘制（四格漫画）
4. 对话合成（自动添加气泡）

### 技术栈
- **后端：** Node.js + Express + TypeScript
- **前端：** Vue 3 + Vite + TypeScript
- **AI 模型：** Google Gemini / OpenAI GPT-4 + DALL-E / Imagen
- **图像处理：** Sharp

### 项目规模
- **文件数量：** 38 个源文件
- **代码行数：** ~1,422 行（TypeScript + Vue）
- **文档行数：** ~1,500 行（Markdown）
- **压缩包大小：** 56KB（不含依赖）

## 🎓 从 RedInk 学到的经验

### 借鉴的优秀设计

1. **清晰的工作流**
   - RedInk：输入 → 生成大纲 → 编辑 → 生成图文
   - 我们：输入 → 生成脚本 → 编辑 → 生成漫画

2. **可编辑的中间产物**
   - RedInk：允许用户编辑大纲
   - 我们：允许用户编辑脚本

3. **详细的错误处理**
   - RedInk：根据错误类型提供具体建议
   - 我们：实现了类似的错误分类和提示

4. **灵活的配置系统**
   - RedInk：YAML 文件管理多个 AI 服务商
   - 我们：.env 文件（更符合 Node.js 生态）

### 我们的创新与差异化

1. **角色一致性**
   - RedInk 是图文内容，我们是漫画
   - 需要同一角色贯穿始终
   - 引入"角色设定图"机制

2. **对话气泡合成**
   - RedInk 的图文是分离的
   - 我们将对话直接合成到图片上
   - 更符合漫画的阅读习惯

3. **四格漫画布局**
   - 2x2 网格展示
   - 经典的四格漫画形式
   - 视觉冲击力强

4. **故事结构优化**
   - Prompt 特别强调"起承转合"
   - 确保四格漫画有完整的叙事弧线

## 🚀 如何使用

### 方式一：快速启动（推荐）

```bash
# 1. 解压项目
tar -xzf comic-ai.tar.gz
cd comic-ai

# 2. 配置 API 密钥
vim backend/.env  # 填入 GOOGLE_API_KEY

# 3. 安装依赖
cd backend && npm install
cd ../frontend && pnpm install

# 4. 启动服务
cd .. && ./start.sh
```

访问：`http://localhost:5173`

### 方式二：Docker 部署

```bash
# 1. 配置环境变量
cp backend/.env.example backend/.env
vim backend/.env  # 填入 API 密钥

# 2. 启动服务
docker-compose up -d
```

访问：`http://localhost:3001`

### 可能的问题与回答

**Q: 为什么选择 Node.js 而不是 Python？**
A: 因为我更擅长前端开发，使用 Node.js 可以实现 TypeScript 全栈，提升开发效率。同时，Node.js 的异步特性非常适合并发图像生成。

**Q: 如何保证角色一致性？**
A: 我们先生成一张"角色设定图"，然后在生成每个分镜时都在 Prompt 中引用这个角色的详细描述。这是借鉴 RedInk 的"参考图"思路，并应用到角色设计上。

**Q: 如果 AI 生成的内容不满意怎么办？**
A: 用户可以在生成图片前编辑脚本，修改画面描述和对话。这是从 RedInk 学到的最重要的产品设计经验。

**Q: 未来如何扩展？**
A: 短期可以支持更多分镜、多角色、风格选择；中期可以加入社区分享、AI 导演模式；长期可以支持语音合成、动画生成，甚至商业化。
