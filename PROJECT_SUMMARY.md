## ✨ 核心特性

### 1. 智能编剧系统
- 一句话创意 → 完整四格漫画脚本
- 结构化输出（标题、角色、分镜、对话）
- 支持 GPT-4 和 Gemini 两种模型

### 2. 角色一致性方案
- 先生成"角色设定图"
- 所有分镜引用同一角色设定
- 解决 AI 生成漫画的最大痛点

### 3. 可编辑工作流
- 用户可以在生成图片前编辑脚本
- 提升用户控制感和满意度
- 借鉴 RedInk 项目的优秀设计

### 4. 对话气泡自动合成
- 使用 Sharp 图像处理库
- 自动将对话文字叠加到图片上
- SVG 矢量图形确保清晰度

### 5. 并发图像生成
- 四个分镜同时生成
- 大幅缩短等待时间（从 120 秒到 30 秒）

## 🎓 学习与借鉴

### 从 RedInk 项目学到的经验

1. **清晰的工作流设计**
   - 输入 → 生成草稿 → 用户编辑 → 最终生成
   - 这个流程非常成熟，我们完全借鉴

2. **灵活的配置系统**
   - 支持多个 AI 服务商
   - 易于切换和管理

3. **详细的日志和错误处理**
   - 快速定位问题
   - 用户友好的错误提示

4. **前后端分离架构**
   - 便于独立开发和部署
   - 提升开发效率

### 我们的创新与差异化

1. **角色一致性方案**
   - RedInk 是图文内容，我们是漫画
   - 需要同一角色贯穿始终
   - 引入"角色设定图"机制

2. **对话气泡合成**
   - RedInk 的图文是分离的
   - 我们将对话直接合成到图片上
   - 更符合漫画的阅读习惯

3. **四格漫画专属布局**
   - 2x2 网格展示
   - 经典的四格漫画形式
   - 视觉冲击力强

4. **故事结构优化**
   - Prompt 特别强调"起承转合"
   - 确保四格漫画有完整的叙事弧线

## 🚀 如何运行

### 最快方式（5 分钟）

```bash
# 1. 配置 API 密钥
vim backend/.env  # 填入 GOOGLE_API_KEY

# 2. 安装依赖
cd backend && npm install
cd ../frontend && pnpm install

# 3. 启动服务
cd .. && ./start.sh
```

### Docker 方式

```bash
docker-compose up -d
```
